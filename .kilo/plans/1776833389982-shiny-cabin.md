PROJECT OVERVIEW:
* A two-repository system separating reusable GitHub Actions from the underlying functional execution code.

THE PROBLEM IT ADDRESSES:
* Monolithic repositories often entangle CI/CD workflow definitions with the actual application logic and scripts.

THE APPROACH TO SOLVING THE PROBLEM:
* Decoupling workflow triggers in a public repository from the functional code in a separate repository.

INSTALLATION:
* Reference the public Action repository in your workflow file using the standard uses syntax.
* Clone the secondary code repository if local execution or modification is required.

USAGE:
* Trigger the GitHub Action from the primary repository via push, pull request, or schedule.
* The Action automatically fetches and executes the associated code from the secondary repository.

EXAMPLES:
* Running automated tests where the Action definition and test scripts live in different version-controlled locations.
* Deploying infrastructure using a centralized public Action that calls dedicated external deployment scripts.
